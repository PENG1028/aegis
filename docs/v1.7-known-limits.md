# Aegis v1.7 Known Limits

## Functional Limits

| Limit | Impact | Mitigation |
|-------|--------|------------|
| **No multi-node control plane** | Aegis runs on one node; follower nodes not supported | Not needed for current single-node deployment |
| **No HMAC dynamic signing** | Gateway Link uses static shared token, not HMAC | Acceptable for personal use; documented in gateway-link-verification.md |
| **Rotate E2E requires manual sync** | After rotate, Server B's verifier must be manually updated with the new token | Documented in gateway-link-rotation.md |
| **No multi-admin** | Only one admin user; no RBAC | Within scope (personal use) |
| **No UI** | All operations via CLI/API | Within scope |
| **No SaaS / multi-tenant** | No tenant isolation, no billing | Within scope |
| **No automatic binary upgrade** | Must manually replace binary and restore DB snapshot | Rollback runbook exists |
| **No deployment executor** | No canary, staged rollout, or scheduler | Beyond scope |
| **No structured log protocol** | No OTel, no log shipping | operation/apply/audit logs exist but no integration |

## Infrastructure Limits

| Limit | Impact | Workaround |
|-------|--------|------------|
| **Cross-VPC traffic blocked by cloud SG** | Two servers in different VPCs cannot communicate on private IPs | Use public IPs with security group rules allowing specific source IPs |
| **No VPC peering** | Server A private IP (10.3.0.4) cannot reach Server B private IP (10.3.0.11) | Required for private routing; currently using public IPs |
| **No VPN/tunnel between servers** | No encrypted channel between control plane and remote targets | Not currently required; traffic uses public internet |
| **Port 80/443 only** | Cloud security group only allows ports 80 TCP and 443 TCP+UDP | All gateway traffic must use these ports |
