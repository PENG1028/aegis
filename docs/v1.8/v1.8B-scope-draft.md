# v1.8B Scope Draft — Route Path Safety & Secret Hardening

> **Theme:** Route safety awareness → optional policy enforcement, plus cryptographic
> hardening for secrets at rest.
>
> **Status:** DRAFT — subject to change before implementation begins.

---

## Design Principles

1. **Policy enforcement is opt-in** — existing v1.8A detection-only behavior unchanged
2. **Secrets never stored in plaintext** — all auth tokens/secrets must be hashed or encrypted at rest
3. **No new provider types** — Caddy/HAProxy only
4. **No OS-level manipulation** — iptables/nftables/eBPF still out of scope
5. **No UI, no monitoring loop, no alert system**

---

## Candidate Work Items

### 1. Route Internal Access Policy (Medium)

Allow routes to declare an internal-only policy that warns/blocks if the target
is external without authorization.

**Scope:**
- Model: `route_internal_policy` flag on routes (internal_only, allow_external)
- Safety check: if `internal_only` + public target → error (not just warning)
- Planner: optional blocking mode for `internal_only` routes
- API: PATCH route to set policy
- CLI: `aegis route update <id> --policy internal-only`

**Not in scope:**
- Per-IP allowlist/denylist
- Geo-based restrictions
- Temporal restrictions

### 2. Gateway Link Required Policy (Small)

Formalize the `gateway_link_required` field from a safety output into an
enforceable policy on routes.

**Scope:**
- Model: `gateway_link_required` boolean on routes (already exists as safety output)
- Apply: if `gateway_link_required=true` and route has no GatewayLink → block apply
- API: PATCH route to set `enforce_gateway_link`
- CLI: `aegis route update <id> --require-gateway-link`

**Not in scope:**
- Auto-generate GatewayLink
- GatewayLink health probe

### 3. Source IP Allowlist Model (Small/Medium)

Allow routes to restrict upstream access to specific source IP ranges.

**Scope:**
- Model: `source_ip_allowlist` JSON array on routes (CIDR notation)
- Caddy render: emit `allowed_networks` in reverse_proxy config (Caddy v2 supports this)
- HAProxy render: emit `allow` ACL in haproxy config
- Safety check: no wildcard `0.0.0.0/0` warning
- API: PATCH route to set allowlist
- CLI: `aegis route update <id> --allowlist 10.0.0.0/8,192.168.0.0/16`

**Not in scope:**
- GeoIP
- DNS-based allowlisting
- Dynamic allowlist updates

### 4. Secret-at-Rest Hardening (Medium)

Cryptographic hardening for secrets stored in the database.

**Scope:**
- **GatewayLink auth_value**: currently stored as plaintext (via `gw.AuthValue`).
  Change to HMAC-SHA256 hash (like admin passwords) + return raw secret once at creation.
- **Migration**: add migration to re-hash existing secrets
- **API**: GET gateway-link no longer returns `auth_value`
- **CLI**: `aegis gateway-link show` no longer shows secret

**Not in scope:**
- Full-database encryption
- HSM integration
- Key rotation service

### 5. Route Safety Warning → Optional Policy Enforcement (Small)

Allow operators to promote a safety warning into a blocking policy.

**Scope:**
- Model: `safety_policy` field on routes (`detect_only`, `warn`, `block`)
- Planner: if `block` + risk with severity >= warning → fail apply
- API: PATCH route safety_policy
- CLI: `aegis route update <id> --safety-policy block`
- Default: `detect_only` — unchanged from v1.8A

**Not in scope:**
- Global default policy
- Policy inheritance

## Out of Scope (v1.8B)

| Item | Reason |
|------|--------|
| ❌ Transparent egress interception | Would violate detection-first principle |
| ❌ Service mesh / sidecar | Too invasive for personal gateway |
| ❌ Full observability system | Deferred to v1.8C |
| ❌ UI | No UI in any v1.x |
| ❌ Multi-admin / SaaS | Personal gateway only |
| ❌ iptables / nftables | OS-level, out of scope |
| ❌ eBPF | Too invasive |
| ❌ Background monitoring loop | Deferred |
| ❌ Alert system | Deferred |
| ❌ Billing / quota | Never planned |
