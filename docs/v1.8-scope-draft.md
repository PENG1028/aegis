# v1.8 Scope Draft: Personal Production Hardening

## Theme

**Egress Trace & Route Path Safety**

v1.8 shifts from "can we make it work" to "can we make it safe and observable".

---

## Proposed Phases

### v1.8A: Egress Trace

Understand where traffic actually goes.

- Target connectivity liveness monitoring (not just trace-time check)
- Periodic health check on targets with event logging
- Trace output enhancement: real-time vs. last-known status
- Route → endpoint → target chain visualization in trace
- Target fail detection with configurable thresholds

### v1.8B: Route Path Safety

Prevent accidental exposure and data leaks.

- Route validation: warn if target_host is public IP without GatewayLink
- Forbid binding to arbitrary public IPs unless explicitly allowed (admin toggle)
- GatewayLink secret hashing in DB (HMAC instead of plaintext)
- GatewayLink attach/detach admin endpoints
- Route change audit trail with diff logging
- `gateway_link_id` must exist before binding (validate link existence)

### v1.8C: Observability Increment

Make production issues findable without code changes.

- Structured log format (JSON log output option)
- Log level configuration (debug/info/warn/error)
- Periodic provider health check with alert events
- Target connectivity drift detection (expected vs. actual)
- operation/apply/audit log retention policy

### v1.8D: Operational Reliability

- Startup self-check (validate config, DB connectivity, provider health)
- Pre-apply safety check (validate no dangerous route changes)
- Apply rollback automation (automatic rollback on reload failure with retry)
- Graceful shutdown signal handling (complete pending operations)
- Periodic SQLite WAL checkpoint / VACUUM

---

## Out of Scope for v1.8

| Item | Reason |
|------|--------|
| Multi-node control plane | Requires consensus, beyond personal use scope |
| UI / SaaS / multi-admin | Not needed |
| New providers (non-Caddy/HAProxy) | Not requested |
| Cloudflare / tunnel automation | Not requested |
| Deployment executor / canary / staged rollout | Not needed |
| HMAC dynamic signing for Gateway Link | Caddy module dependency, deferred |
| Real multi-node cluster | real_env_required |
